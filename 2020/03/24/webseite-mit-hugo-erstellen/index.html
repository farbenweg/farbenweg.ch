Webseite mit Hugo erstellen

24.03.2020

Das ist mein erster Post. Heute ist ein sonniger Märztag mit einer kalten Bise. Ich lerne es, Webseiten mit Hugo zu erstellen.

Dazu benutze ich die drei Komponenten Hugo, Windows PowerShell ISE, Visual Studio Code und Github.

Eine Seite erstellen

In Windows PowerShell gehe ich ins Verzeichnis \farbenweg>. Dort erstelle ich den Ordner blog-lithium und eine neue Hugo Seite:

md blog-lithium
hugo new site ./

Automatisch sind die Ordner archetypes, content, data, layouts, resources, static und themes und die Datei config.toml entstanden. Als zweites passe ich die config.toml an:

baseURL = "http://localhost:1313/"
languageCode = "de-de"
title = "Farbenweg"

Eine leere Seite ist generiert.

Ein Theme herunterladen

Um ein Theme zu installieren, wechsle ich ins entsprechende Verzeichnis.

cd themes
git clone https://github.com/jrutheiser/hugo-lithium-theme
cd ../

Das Theme kann in lithium umbenannt und in der Konfigurationsdatei config.toml als solches festgelegt werden:

baseURL = "http://localhost:1313/"
languageCode = "de-de"
title = "Beispiel Hugo"
theme = "lithium"

Dann kopiere ich die Inhalte von ExampleSite in mein Verzeichnis.

Einen Inhalt erstellen

Um einen ersten Blogeintrag zu erstellen, verwende ich den Befehl hugo new post/___.md:

hugo new post/Webseite-mit-Hugo-erstellen.md

Es entsteht im Unterverzeichnis content/post/ eine Seite, in die ich einen Eintrag schreiben kann:

---
title: "Webseite mit Hugo erstellen"
date: 2020-03-24T09:51:12+01:00
draft: true
---
Das ist mein erster Post. Heute ist ein sonniger Märztag mit einer kalten Bise.

Jetzt entdecke ich Markdown mit seinen unterschiedlichen Formatierungen. Desweiteren möchte ich jetzt ein Bild einfügen. Ich lege es in das Verzeichnis static/images.

Um die Website anzuschauen starte ich

hugo -D serve

und gehe auf http://localhost:1313/. Die Foto wird angezeigt.

Auf Domain ziehen

Um die Website auf meiner Github Domain anzuzeigen, gehe ich folgendermassen vor:

rmdir public
git clone https://github.com/farbenweg/farbenweg.github.io public
hugo
cd public
git add --all
git commit -m "inital commit"
git push origin master

Dann versuche ich https://farbenweg.github.io/ und es ist nur ein unformatierter Text zu sehen. Es ist nicht das erste Mal, dass ich enttäuscht abbreche. Doch dann ändere ich zwei Dinge: Die Posts werden auf

draft: false

gestellt. Und die config.toml wird auf eine andere baseURL gelegt:

baseURL = "https://farbenweg.github.io/"

Es klappt nicht gleich. Ein paar Minuten später sehe ich auf meinem Smartphone, dass die Posts jetzt alle da sind! Es braucht einfach etwas Zeit, bis sie auf der Domain ankommen.

In config.toml ändere ich die Menüeinträge:

[[menu.main]]
    name = "Farbenweg"
    url = "/farbenweg/"
[[menu.main]]
    name = "Kontakt"
    url = "/kontakt/"
[[menu.main]]
    name = "Über"
    url = "/ueber/"

In content/page/ erstelle ich die drei Einträge für die Seiten: Farbenweg.md, Atelier.md und Ueber.md. Diese Seiten können mit Inhalt gefüllt werden.

Das Über funktioniert

Das war eine längere Sache. Schon seit letztem August fragte ich mich, ob in Hugo deutsche Page-Titel funktionieren. Vor allem für die Über-Seite war es mir wichtig. Die Lösung habe ich heute hier gefunden:

https://discourse.gohugo.io/t/make-url-use-only-ascii-characters/7592

Das heisst, es braucht nur eine Zeile in der config.toml:

RemovePathAccents = true

Im Titel ist jetzt ein Über anstelle von Ueber zu sehen! In der URL ist einfach ein u anstelle des ü:

https://farbenweg.github.io/uber.html

Und entsprechend sieht es für den Titel dieses Posts aus. Sehr schön, dass die Lösung so einfach war.

Neu auf Github laden

Nach längerem Testen haben sich im Github-Ordner eine Menge angesammelt, die nicht mehr gebraucht werden. Deshalb möchte ich diese Daten löschen. Wenn ich das Theme wechsle, gehe ich ebenfalls so vor, um das Repository ganz zu leeren. Ich wechsle in den Ordner: farbenweg\blog-lithium\public [master ≡]>

git pull
git rm -r .
git commit -m "leer"
git push

Jetzt ist das Repository leer und ich kann die aktuelle Seite hochladen.

Auf andere Domain laden

Ich kopiere die bisherige Seite in meinen public-Ordner.

git clone https://github.com/farbenweg/farbenweg.ch public

Eventuell lösche ich die alte Seite zuerst:

cd public
git rm -r .
git commit -m "leer"
git push

Dann kompiliere ich die Seite und ziehe sie auf die Domain:

git clone https://github.com/farbenweg/farbenweg.ch public
hugo
cd public
git add --all
git commit -m "inital commit"
git push origin master

Jetzt sind alle Dateien auf der Domain. Leider ist nichts im Browser zu sehen. Ich habe die Datei CNAME vergessen. Sie enthält den Domainnamen, wenn ich eine beliebige Domain wähle:

farbenweg.ch

Nun funktioniert das auf der Domain farbenweg.ch. Dort arbeite ich für den Moment weiter.

Favicon

Seit drei Tagen übe ich mich an den Icons der Webseite. Ich habe gesehen, dass es zwei Arten von Icons gibt. Die ganz kleinen nennen sich favicon.ico oder favicon.png und befinden sich oben im Fenster oder im Tab. Sie sind 16 x 16 Pixel, 32 x 32 Pixel oder 48 x 48 Pixel klein:

favicon-32x32.png

favicon.ico
